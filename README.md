# 🎬 AI Video Summarizer & Intelligence Platform

An advanced, full-stack AI application that transforms long videos into concise, high-impact summaries. Using **Groq's LLaMA 3.3 70B** and **Whisper-large-v3**, this platform intelligently analyzes transcripts to generate a visual summary video that is strictly **40-50%** of the original duration.

---

## 🚀 How it Works (The Pipeline)

1.  **Video Upload**: Users upload a local video or provide a YouTube URL.
2.  **Audio Extraction**: The backend uses `ffmpeg` to extract high-quality audio.
3.  **Fast Transcription**: Audio is sent to **Groq Whisper** for near-instant speech-to-text conversion.
4.  **AI Analysis**: 
    *   **Summarization**: LLaMA 3.3 generates a text summary and key points.
    *   **Ranking**: The AI scores every segment of the video based on information density.
5.  **Visual Synthesis**: **MoviePy** identifies the highest-ranked clips and stitches them together to create a summary video that fits within the **40-50% duration constraint**.

## 📂 Project Structure & File Explanations

### 🧠 Backend (FastAPI)
*   `app/main.py`: The entry point. Initializes the FastAPI app and loads the AI models into memory.
*   `app/api/summarize.py`: The "Brain." Coordinates the entire async pipeline from transcription to video generation.
*   `app/models/video_processor.py`: The "Editor." Contains the MoviePy logic for sub-clipping, stitching, and duration management.
*   `app/models/summarizer.py`: The "Thinker." Interfaces with Groq LLMs to rank segments and summarize content.
*   `app/models/whisper_model.py`: Handles high-speed transcription via the Groq Cloud API.
*   `app/core/constants.py`: Stores global configuration like the **40-50% summary ratio** and model names.

### 💻 Frontend (React)
*   `src/components/`: Contains UI elements like the Video Player, Upload Zone, and Chat Interface.
*   `src/pages/`: Main views including the Dashboard and the Summary Results page.
*   `src/hooks/`: Custom React hooks for managing API calls and real-time progress polling.

## 🛠️ Setup & Installation

### Prerequisites
*   Python 3.10+
*   Node.js 18+
*   MongoDB (Running locally or via Atlas)
*   **Groq API Key** (Get one at [console.groq.com](https://console.groq.com))

### 1. Backend Setup
```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```
Create a `.env` file in the `backend/` directory:
```env
GROQ_API_KEY=your_key_here
MONGODB_URL=mongodb://localhost:27017
SECRET_KEY=your_random_secret
```
Run the server:
```bash
python -m uvicorn app.main:app --reload --port 8000
```

### 2. Frontend Setup
```bash
cd frontend
npm install
```
Run the development server:
```bash
npm start
```
The app will be available at `http://localhost:3000`.

## 🔍 Technical Deep Dive

### 1. The 40-50% Duration Logic
A core requirement of this project is to ensure the summary is neither too short (losing information) nor too long (losing the benefit of a summary). 
*   **The Algorithm**: We use a **Greedy Selection Algorithm** combined with **Heuristic Ranking**.
*   **Methodology**: 
    1.  Calculate `Target_Min = Original_Duration * 0.40` and `Target_Max = Original_Duration * 0.50`.
    2.  Sort all transcribed segments by their **AI Relevance Score** (generated by LLaMA 3.3).
    3.  Iterate through the sorted list, adding the "most important" segments to our "Summary Budget" until we fall within the 40-50% window.
    4.  If a segment would push us over 50%, the system mathematically trims its `end_time` to fit the budget exactly.

### 2. Why Groq Cloud?
Traditional local AI models (like running Whisper on a standard laptop) can take 5-10 minutes to transcribe a 2-minute video. 
*   **Groq LPU Architecture**: By using Groq's Language Processing Units, we achieve inference speeds of **~500 tokens per second**.
*   **Efficiency**: This allows our "Summarization" step to complete in under 2 seconds, providing a "Real-Time" feel to the user.

## 🗄️ Database Schema (MongoDB)

The project uses three main collections to ensure data persistence:

1.  **Users**: Stores hashed passwords and user profiles.
2.  **Videos**: Tracks uploaded file paths, original metadata (size, duration), and processing status.
3.  **Summaries**: The central hub. Stores:
    *   `text_summary`: The generated paragraphs.
    *   `key_points`: Array of strings.
    *   `segments`: Array of objects containing `start`, `end`, and `text`.
    *   `summary_video_path`: The physical location of the generated MP4.

## 🛠️ Troubleshooting & Common Fixes

*   **MoviePy Versions**: This project is optimized for **MoviePy 2.1.2**. If you use version 1.x, the `subclipped()` and `resized()` methods will fail. We have included a compatibility layer in `video_processor.py`.
*   **FFMPEG**: Ensure `ffmpeg` is installed on your system path. Without it, audio extraction and video stitching will fail.
*   **Numpy Compatibility**: If you see `ImportError: multiarray failed to import`, ensure your `numpy` version is `1.26.4` to maintain compatibility with `moviepy` and `numba`.
